# failure-owner-should-switch-earlier

## Trigger
- Evidence keeps arriving from a supporting surface, but the current primary owner still frames the solve.

## Why it looked reasonable
- The visible UI or original route made the current owner feel natural.

## Misleading signal
- Transport-level visibility was mistaken for sink ownership.

## Earlier kill signal
- The supporting surface explained the parser, sink, runtime, or closure blocker better than the current owner.

## Better next probe
- Write an Owner Matrix and perform a deliberate handoff with a named closure owner.

## Stop rule
- No co-primary owners; if support owns sink/closure understanding, hand it the branch.

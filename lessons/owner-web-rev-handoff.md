# owner-web-rev-handoff

## Trigger
- The challenge is delivered through HTTP/UI/API, but downloaded clients, scripts, APKs, binaries, or frontend logic reveal transforms, secrets, validation, or protocol semantics.

## Why it looks promising
- Web exposes the workflow, but Rev may explain the true checker, token format, local encryption, or protocol state.

## What usually goes wrong
- Web remains owner even after the real semantic problem has become reverse-engineering a client or downloaded artifact.

## Better question
- Is HTTP only transport now, while the actual sink, transform, or trust boundary is owned by the artifact logic?

## Handoff trigger
- Artifact logic explains the token/secret/validation/serialization state better than black-box Web mutation does.

## Return trigger
- Return to Web once the artifact-derived insight produces a concrete route, payload, or auth/session path to test.

## Closure owner
- The owner that controls primitive-to-flag completion after the artifact insight is applied.

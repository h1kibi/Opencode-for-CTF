# owner-web-crypto-handoff

## Trigger
- A Web challenge exposes tokens, signatures, encrypted blobs, oracle-like responses, or protocol verification behavior.

## Why it looks promising
- Web owns the route, but Crypto may own the actual trust boundary and exploit primitive.

## What usually goes wrong
- The solve keeps mutating Web requests while the key weakness is scheme structure, oracle behavior, or parameter misuse.

## Better question
- Is the main uncertainty now route behavior, or cryptographic semantics?

## Handoff trigger
- The next decisive probe depends more on decoding, modeling, or breaking scheme behavior than on Web transport differences.

## Return trigger
- Return to Web once the crypto result yields a forged token, plaintext, role, credential, or request field that must be exercised through the app.

## Closure owner
- The owner that carries the forged/decrypted capability to the final flag path.
